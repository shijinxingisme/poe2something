# POE2 市场套利分析器

这是一个 POE2 通货市场套利分析 Web 应用。它会读取 poe2db Economy 行情快照，也可以在页面里手动刷新物价，并支持你录入实际交易报价，扫描 A→B→C→A 三步闭环路径，按收益率展示 Top N 套利机会。

## 核心功能

- 默认套用 poe2db Economy 快照价格
- 页面提供“刷新物价”按钮，使用时再尝试读取 poe2db 最新行情
- 维护通货价格表，价格统一折算为混沌石
- 支持从 poe2db Economy 页面复制通货行并手动导入价格
- 支持手动录入实际交易报价，例如：`混沌石>崇高石=0.052`
- 扫描 A→B→C→A 闭环路径
- 展示投入、回收、收益率和使用到的三段报价
- 支持设置起点通货、投入数量、Top N 和最低收益率
- 淘宝模式默认关闭，开启后低频检查你提供的集市地址，发现低于目标价时发出页面和浏览器通知

## 淘宝模式单位

淘宝模式的目标价格需要选择单位。页面会把目标价和识别到的集市价格统一折算成混沌石后比较。

例如：目标价填 `1`，单位选 `神圣石`。如果集市里识别到 `100E`，页面会按当前物价把 `100 崇高石` 折算成混沌石；只要它低于 `1 神圣石` 的混沌石价值，就会提醒。

如果集市页面没有写单位，页面会使用“未识别单位时按”这个选项。

## 数据来源

行情来源：<https://poe2db.tw/cn/Economy>

静态网页会先读取仓库中的 `poe2db-prices.js` 快照。点击“刷新物价”时，页面会尝试从 poe2db 获取最新行情；如果浏览器跨站限制导致失败，可以打开 poe2db 后复制行情到页面里导入。

## 淘宝模式说明

淘宝模式需要你提供集市网页地址、目标价格和刷新间隔。刷新间隔最低限制为 60 秒，避免过于频繁请求。

如果目标网页需要登录凭证，可以在 Token / Cookie 区域粘贴请求头。浏览器网页可以发送 `Authorization` 这类请求头，但不能可靠手动发送 Cookie；如果目标站点依赖 Cookie 或限制跨站读取，后续需要使用你自己可控的代理服务。

实际套利判断建议以游戏内和交易站即时挂单为准。
